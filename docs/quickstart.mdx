## Prerequisites

- Node.js 20+
- Claude Code (or Cursor / Windsurf)
- A project to index

## 1. Install

```bash
npm install -g @phmotad/fire-code
```

## 2. Install the Plugin

<Tabs>
  <Tab title="Claude Code">
    ```bash
    fire-code install
    ```
    This registers the MCP server and lifecycle hooks in your Claude Code config.
  </Tab>
  <Tab title="Cursor">
    ```bash
    fire-code install --ide cursor
    ```
  </Tab>
  <Tab title="Windsurf">
    ```bash
    fire-code install --ide windsurf
    ```
  </Tab>
</Tabs>

## 3. Index Your Project

Navigate to your project and run:

```bash
cd your-project
fire-code index
```

This scans your source files, builds the dependency graph, and generates semantic embeddings. Stored in `.firecode/` — never committed to git.

## 4. (Optional) Build the Knowledge Corpus

If your project has architecture docs, ADRs, or READMEs:

```bash
fire-code corpus build
```

This indexes all `.md`, `.txt`, `.rst`, and `.mdx` files into a full-text search database.

## 5. Restart Claude Code

The MCP tools are now available in every session. Claude will automatically inject the index summary on session start.

## Verify It's Working

In Claude Code, ask:

```
Use firecode.smart_search to find the main entry point of this project.
```

You should get a response with file and line references from your codebase.

## What Happens on Each Session

```
SessionStart hook fires
  ├── Daemon starts (port 37778) if not already running
  └── .firecode/bootstrap.log injected into context
        → Claude knows: N files indexed, M functions, last indexed at T

PostToolUse hook fires (on Write/Edit)
  └── Changed file re-indexed automatically
```
