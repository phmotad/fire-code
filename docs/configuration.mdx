Create `firecode.config.ts` in your project root to customize Fire Code. Run `fire-code init` for an interactive setup wizard.

## Full Config Reference

```typescript
import type { FireCodeConfig } from '@phmotad/fire-code';

export default {
  project: {
    name: 'my-project',       // used for DB namespacing
    mode: 'standard',         // 'standard' | 'legacy'
  },

  llm: {
    provider: 'anthropic',    // 'anthropic' | 'openai' | 'openrouter' | 'ollama'
    model: 'claude-sonnet-4-6',
    apiKey: process.env.ANTHROPIC_API_KEY,
    maxTokens: 4096,
    temperature: 0.2,
  },

  embeddings: {
    provider: 'local',        // 'local' only (runs offline via @xenova/transformers)
    model: 'Xenova/all-MiniLM-L6-v2',
  },

  git: {
    branchStrategy: 'increment',  // 'reuse' | 'increment' | 'fail'
    workingTree: 'stash',         // 'stash' | 'commit' | 'fail' | 'ignore'
    autoCommit: true,
    commitFormat: 'conventional', // conventional commit format
  },

  indexing: {
    mode: 'lazy',             // 'lazy' | 'full'
    include: ['**/*.ts', '**/*.tsx', '**/*.js', '**/*.jsx'],
    exclude: ['node_modules', 'dist', 'build', '.git', 'coverage'],
  },

  execution: {
    conflictStrategy: 'merge',    // 'merge' | 'overwrite' | 'fail'
    dryRun: false,
    validateSyntax: true,
  },
} satisfies FireCodeConfig;
```

## LLM Providers

| Provider | `provider` value | Required env var | Default model |
|----------|-----------------|-----------------|---------------|
| Anthropic | `anthropic` | `ANTHROPIC_API_KEY` | `claude-sonnet-4-6` |
| OpenAI | `openai` | `OPENAI_API_KEY` | `gpt-4o` |
| OpenRouter | `openrouter` | `OPENROUTER_API_KEY` | `deepseek/deepseek-coder` |
| Ollama | `ollama` | _(none)_ | `codellama` |

## Git Strategies

### Branch Strategy (`git.branchStrategy`)

| Value | Behavior |
|-------|----------|
| `reuse` | Check out the branch if it already exists |
| `increment` | Add `-2`, `-3`... suffix if branch exists |
| `fail` | Throw `GitError` if branch exists |

### Working Tree Strategy (`git.workingTree`)

| Value | Behavior |
|-------|----------|
| `stash` | Automatically run `git stash` before execution |
| `commit` | Auto-commit dirty files with a "WIP" message |
| `fail` | Block execution if there are uncommitted changes |
| `ignore` | Proceed regardless of dirty state |

## Config Loading Order

Fire Code looks for config in this order:

1. `./firecode.config.ts`
2. `./firecode.config.json`
3. `~/.firecode/config.ts`
4. Built-in defaults

## Project Mode

### `standard` (default)
For greenfield projects or well-organized codebases. Uses strict graph traversal and full embedding generation.

### `legacy`
For large, messy, or partially-typed codebases. Uses fallback text search when graph/vector retrieval fails. Recommended if you have files without TypeScript types or non-standard module patterns.

<Tip>
  Run `fire-code init` for an interactive wizard that creates `firecode.config.ts` with the right settings for your project.
</Tip>
