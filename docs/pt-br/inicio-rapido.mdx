description: Instale o Fire Code e comece a usá-lo em menos de 5 minutos.
---

## Pré-requisitos

- Node.js 20+
- Claude Code, Cursor, Windsurf, OpenCode ou outro IDE com suporte a MCP
- Um projeto para indexar

## 1. Instalar

```bash
npm install -g @phmotad/fire-code
```

## 2. Instalar o Plugin

<Tabs>
  <Tab title="Claude Code">
    ```bash
    fire-code install
    ```
    Registra o servidor MCP, hooks de ciclo de vida e escreve `agents.md` na raiz do projeto.
  </Tab>
  <Tab title="Cursor">
    ```bash
    fire-code install --ide cursor
    ```
  </Tab>
  <Tab title="Windsurf">
    ```bash
    fire-code install --ide windsurf
    ```
  </Tab>
  <Tab title="OpenCode">
    ```bash
    fire-code install --ide opencode
    ```
  </Tab>
  <Tab title="Outros">
    ```bash
    fire-code install --ide generic
    ```
    Detecta o IDE automaticamente se nenhuma flag for passada.
  </Tab>
</Tabs>

O `fire-code install` também cria um arquivo `agents.md` na raiz do projeto. Esse arquivo contém as regras de disparo que dizem ao agente **quando invocar cada ferramenta MCP automaticamente** — sem nenhuma intervenção sua.

## 3. Indexar o Projeto

Navegue até o projeto e execute:

```bash
cd seu-projeto
fire-code index
```

Escaneia os arquivos fonte, constrói o grafo de dependências e gera embeddings semânticos. Tudo salvo em `.firecode/` — nunca commitado no git.

## 4. (Opcional) Construir o Corpus de Conhecimento

Se o projeto tem docs de arquitetura, ADRs ou READMEs:

```bash
fire-code corpus build
```

Indexa todos os arquivos `.md`, `.txt`, `.rst` e `.mdx` em um banco de busca full-text, para que o agente consulte decisões arquiteturais antes de escrever código.

## 5. Reiniciar o IDE

O Fire Code está ativo. Na próxima sessão, o agente recebe o resumo do índice automaticamente e começa a usar as ferramentas MCP por conta própria.

## Como Saber que Está Funcionando

Abra o projeto e inicie uma tarefa de codificação — **não peça ao agente para usar nenhuma ferramenta manualmente**. Observe o agente invocar as ferramentas Fire Code automaticamente antes de fazer qualquer alteração:

```
→ firecode.smart_search({ query: "..." })     ← busca antes de escrever
→ firecode.corpus_search({ query: "..." })    ← verifica restrições arquiteturais
→ firecode.execute({ task: "...", agent: "dev" })  ← cria branch e commita
```

O arquivo `agents.md` na raiz do projeto é o que dirige esse comportamento. Se o agente não estiver usando as ferramentas automaticamente, verifique se o `agents.md` existe e se o IDE foi reiniciado após a instalação.

## O que Acontece em Cada Sessão

```
Hook SessionStart dispara
  ├── Daemon inicia (porta 37778) se ainda não estiver rodando
  └── .firecode/bootstrap.log injetado no contexto
        → Agente sabe: N arquivos indexados, M funções, última indexação em T

Hook PostToolUse dispara (em Write/Edit)
  └── Arquivo alterado re-indexado automaticamente
```
