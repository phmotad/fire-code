<Info>
Você não precisa criar nenhum arquivo de configuração manualmente. Execute `fire-code install` para criar a configuração global automaticamente, ou `fire-code init` para o assistente interativo do projeto.
</Info>

## Onde fica a configuração

O Fire Code armazena configuração em arquivos JSON, mesclados nesta ordem de prioridade:

| Local | Escopo | Criado por |
|-------|--------|-----------|
| `.firecode/config.json` | Projeto (maior prioridade) | `fire-code init` ou `fire-code config set` |
| `~/.firecode/config.json` | Global (fallback) | `fire-code install` |
| Padrões internos | Sempre aplicados | — |

A configuração do projeto substitui a global chave a chave.

## Comandos

Toda a configuração é editável via `fire-code config`:

```bash
# Ver toda a configuração (global + projeto)
fire-code config list

# Ver só a configuração global
fire-code config list --global

# Ler um valor específico
fire-code config get llm.provider

# Definir no projeto (.firecode/config.json)
fire-code config set llm.provider anthropic
fire-code config set llm.model claude-sonnet-4-6
fire-code config set llm.apiKey sk-...

# Definir globalmente (~/.firecode/config.json)
fire-code config set --global llm.provider openrouter
fire-code config set --global llm.apiKey sk-...

# Remover uma chave
fire-code config unset llm.apiKey
fire-code config unset --global llm.apiKey
```

Notação com ponto navega chaves aninhadas: `git.branchStrategy`, `indexing.mode`, `llm.provider`.

## Referência Completa

O arquivo de config é JSON simples:

```json
{
  "project": {
    "name": "meu-projeto",
    "mode": "standard"
  },
  "llm": {
    "provider": "anthropic",
    "model": "claude-sonnet-4-6",
    "apiKey": "sk-...",
    "maxTokens": 4096,
    "temperature": 0.2
  },
  "git": {
    "enabled": true,
    "branchStrategy": "reuse",
    "workingTree": "stash",
    "autoCommit": true,
    "commitFormat": "conventional"
  },
  "indexing": {
    "mode": "lazy",
    "include": ["**/*.ts", "**/*.tsx", "**/*.js", "**/*.jsx"],
    "exclude": ["node_modules/**", "dist/**", ".firecode/**"]
  },
  "execution": {
    "conflictStrategy": "merge",
    "dryRun": false,
    "validateSyntax": true
  }
}
```

## Providers de LLM

| Provider | Valor `provider` | Modelo padrão | Variável de ambiente |
|----------|-----------------|--------------|---------------------|
| Anthropic | `anthropic` | `claude-sonnet-4-6` | `ANTHROPIC_API_KEY` |
| OpenAI | `openai` | `gpt-4o` | `OPENAI_API_KEY` |
| OpenRouter | `openrouter` | `deepseek/deepseek-coder` | `OPENROUTER_API_KEY` |
| Ollama | `ollama` | `codellama` | _(nenhuma)_ |

Se `apiKey` não estiver no config, o provider lê a variável de ambiente correspondente automaticamente.

## Estratégias Git

### `git.branchStrategy`

| Valor | Comportamento |
|-------|--------------|
| `reuse` | Faz checkout do branch se já existir |
| `increment` | Adiciona sufixo `-2`, `-3`... se o branch existir |
| `fail` | Lança `GitError` se o branch existir |

### `git.workingTree`

| Valor | Comportamento |
|-------|--------------|
| `stash` | Executa `git stash` automaticamente antes da execução |
| `commit` | Auto-commita arquivos modificados com mensagem "WIP" |
| `fail` | Bloqueia se houver mudanças não commitadas |
| `ignore` | Prossegue independente do estado do working tree |

## Modo do Projeto

### `standard` (padrão)
Projetos novos ou bem organizados. Usa traversal de grafo estrito e geração completa de embeddings.

### `legacy`
Projetos grandes ou parcialmente tipados. Usa busca textual como fallback quando o grafo/vetor falha. Recomendado para projetos sem tipos TypeScript ou com padrões de módulo não convencionais.
