<Info>
You never need to create a config file by hand. Run `fire-code install` to bootstrap global config automatically, or `fire-code init` for the interactive project wizard.
</Info>

## Config Locations

Fire Code stores configuration in JSON files, merged in this priority order:

| Location | Scope | Created by |
|----------|-------|-----------|
| `.firecode/config.json` | Project (highest priority) | `fire-code init` or `fire-code config set` |
| `~/.firecode/config.json` | Global (fallback) | `fire-code install` |
| Built-in defaults | Always applied | — |

Project config overrides global config on a per-key basis.

## CLI Commands

All configuration is editable through `fire-code config`:

```bash
# Show all config (global + project)
fire-code config list

# Show global config only
fire-code config list --global

# Get a specific value
fire-code config get llm.provider

# Set in project config (.firecode/config.json)
fire-code config set llm.provider anthropic
fire-code config set llm.model claude-sonnet-4-6
fire-code config set llm.apiKey sk-...

# Set in global config (~/.firecode/config.json)
fire-code config set --global llm.provider openrouter
fire-code config set --global llm.apiKey sk-...

# Remove a key
fire-code config unset llm.apiKey
fire-code config unset --global llm.apiKey
```

Dot notation navigates nested keys: `git.branchStrategy`, `indexing.mode`, `llm.provider`.

## Full Config Reference

The config file is plain JSON:

```json
{
  "project": {
    "name": "my-project",
    "mode": "standard"
  },
  "llm": {
    "provider": "anthropic",
    "model": "claude-sonnet-4-6",
    "apiKey": "sk-...",
    "maxTokens": 4096,
    "temperature": 0.2
  },
  "git": {
    "enabled": true,
    "branchStrategy": "reuse",
    "workingTree": "stash",
    "autoCommit": true,
    "commitFormat": "conventional"
  },
  "indexing": {
    "mode": "lazy",
    "include": ["**/*.ts", "**/*.tsx", "**/*.js", "**/*.jsx"],
    "exclude": ["node_modules/**", "dist/**", ".firecode/**"]
  },
  "execution": {
    "conflictStrategy": "merge",
    "dryRun": false,
    "validateSyntax": true
  }
}
```

## LLM Providers

| Provider | `provider` value | Default model | API key env var |
|----------|-----------------|--------------|----------------|
| Anthropic | `anthropic` | `claude-sonnet-4-6` | `ANTHROPIC_API_KEY` |
| OpenAI | `openai` | `gpt-4o` | `OPENAI_API_KEY` |
| OpenRouter | `openrouter` | `deepseek/deepseek-coder` | `OPENROUTER_API_KEY` |
| Ollama | `ollama` | `codellama` | _(none)_ |

If `apiKey` is not set in config, the provider reads the corresponding environment variable automatically.

## Git Strategies

### `git.branchStrategy`

| Value | Behavior |
|-------|----------|
| `reuse` | Check out the branch if it already exists |
| `increment` | Add `-2`, `-3`... suffix if branch exists |
| `fail` | Throw `GitError` if branch exists |

### `git.workingTree`

| Value | Behavior |
|-------|----------|
| `stash` | Automatically run `git stash` before execution |
| `commit` | Auto-commit dirty files with a "WIP" message |
| `fail` | Block execution if there are uncommitted changes |
| `ignore` | Proceed regardless of dirty state |

## Project Mode

### `standard` (default)
Greenfield projects or well-organized codebases. Strict graph traversal and full embedding generation.

### `legacy`
Large or partially-typed codebases. Falls back to text search when graph/vector retrieval fails. Recommended if you have files without TypeScript types or non-standard module patterns.
